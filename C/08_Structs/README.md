# 📂 08. Structs
>**학습 목표:** 구조체(struct)를 선언하고 접근하는 방법 이해, <br>
>포인터를 활용하여 배열의 요소를 수정하는 방법 습득, <br>
>배열과 구조체를 결합하여 여러 데이터를 효율적으로 관리하는 능력 배양.
---
### 📝 오늘 배운 핵심 개념
**1. 구조체의 정의와 선언**
* **구조:** 여러 자료형의 데이터를 하나로 묶는 자료구조. (예: `struct Person { char name[20]; int age; };`)
* **특징:** 서로 다른 타입의 멤버를 하나의 이름으로 관리 가능.
* **멤버 접근:** 점(`.`) 연산자를 사용하여 구조체 변수의 멤버에 접근. (예: `p.name`, `p.age`)

**2. 구조체와 배열의 결합**
* **배열 선언:** `struct Person p[3];` - 3개의 구조체 변수를 배열로 관리
* **멤버 접근:** 배열 인덱스와 점 연산자를 함께 사용. (예: `p[i].name`, `p[i].age`)
* **루프 활용:** `for`문이나 `while`문으로 배열의 모든 구조체 요소에 순차적으로 접근.

**3. 포인터를 이용한 배열 수정**
* **포인터 전달:** 함수의 매개변수로 배열의 주소를 전달. (예: `void doubleArray(int *arr, int size)`)
* **주소 연산:** `*(arr+i)` 형태로 포인터를 통해 배열의 i번째 원소에 접근.
* **값 수정:** `*(arr+i) *= 2;` - 포인터가 가리키는 메모리의 값을 직접 수정.

**4. scanf 사용 시 주의사항**
* **일반 변수:** `&` 기호를 붙여서 주소를 전달. (예: `scanf("%d", &p.age);`)
* **문자 배열:** `&` 기호 **없이** 배열 이름만 전달. (예: `scanf("%s", p.name);`)
* **이유:** 배열 이름은 이미 배열의 첫 번째 주소를 나타내므로 추가로 `&`를 붙이지 않음.

**5. 포인터 표기법과 배열 인덱싱**
* **동등성:** `*(arr+i)` = `arr[i]` (두 표기법은 동일한 결과를 제공)
* **선택:** 코드의 가독성에 따라 둘 중 편한 방식 사용 가능.

### TODAY ISSUE
⚠️ 오늘의 주요 트러블슈팅 (Cautions)
| 구분 | 내용 |
| :--- | :--- |
| **& 기호 오류** | `scanf`에서 문자 배열은 `&`를 붙이면 안 됨. `scanf("%s", &p.name);` ❌ → `scanf("%s", p.name);` ✓ |
| **포인터 연산** | `*(arr+i)`는 배열의 i번째 원소를 의미. `*arr+i`와는 다름 (괄호 위치 중요!). |
| **구조체 배열 초기화** | `for`문으로 배열의 각 원소에 개별적으로 접근하여 값을 입력받아야 함. |
| **max 값 초기화** | 최댓값을 구할 때 `int max=0;`보다는 `int max=p[0].age;`로 초기화하는 것이 정석. |
