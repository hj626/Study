# 📂 06. Strings
>**학습 목표:** 문자 배열(char array)을 통해 문자열을 선언하고 다루는 방법 이해, <br>
>`scanf`와 `fgets`의 차이를 파악, 공백을 포함한 문자열 입력 처리를 숙달. <br>
>반복문을 활용하여 문자열의 길이를 직접 구하는 로직구현.

---

### 📝 오늘 배운 핵심 개념

**1. 문자열의 정의와 선언**
* **구조:** 문자열은 `char` 배열로 선언. (예: `char name[20];`)
* **특징:** 배열 마지막에는 자동으로 **널 문자(`\0`)** 가 붙어 문자열의 끝을 표시. 실제 저장 가능한 문자 수는 `크기 - 1`개.

**2. `scanf` vs `fgets`**
* **`scanf("%s", name)`:** 공백 이전까지만 입력을 받음. 한 단어 입력에 사용.
* **`fgets(name, sizeof(name), stdin)`:** 공백을 포함한 문장 전체를 입력받을 수 있음. 단, **엔터(`\n`)까지 함께 저장**되는 점에 주의.

**3. `sizeof()` vs 숫자 직접 입력**
* `fgets`의 두 번째 인자로 숫자를 직접 쓰면 안 됨.
* `sizeof(50)`은 정수 50의 크기인 **4바이트**로 인식됨.
* **반드시 `sizeof(변수명)` 형태로 작성**해야 배열의 실제 크기(바이트)를 전달할 수 있음.

**4. `stdin` (Standard Input)**
* **표준 입력(키보드)**을 의미하는 상수.
* `fgets`의 세 번째 인자로 사용하여 "키보드로부터 입력을 받겠다"는 것을 명시.

**5. 문자열 길이 구하기**
* 별도 함수 없이 반복문으로 직접 구현 가능.
* `\0`(널 문자)을 만날 때까지 인덱스를 증가시켜 길이를 카운트.
* `for`문과 `while`문 두 가지 방식으로 구현 가능.

⚠️ 오늘의 주요 트러블슈팅 (Cautions)

| 구분 | 내용 |
| :--- | :--- |
| **fgets와 개행 문자** | `fgets`는 엔터(`\n`)까지 문자열에 포함시킴. 문자 3개를 입력해도 길이가 4로 출력되는 원인. |
| **sizeof 인자 오류** | `fgets` 사용 시 `sizeof(50)`처럼 숫자를 직접 넣으면 4바이트로 인식됨. 반드시 `sizeof(배열명)` 사용! |
